# torch is NOT pinned here and is installed SEPARATELY by run.sh from a CUDA wheel index,
# because the right build depends on your GPU driver AND your Python version - a single hard
# pin (e.g. torch==2.5.1 on cu121) has no wheel for, say, Python 3.13 and fails with
# "No matching distribution found". run.sh runs, before this file:
#     pip install torch --index-url $TORCH_INDEX   (default cu124; override for your driver)
# See https://pytorch.org/get-started/locally/ for the index matching your CUDA.
#
# Version strategy (learned the hard way): "install latest of everything" breaks both ways -
#   - libs too OLD for the fresh torch  -> transformers/bitsandbytes built for Triton 2.x hit
#     "No module named 'triton.ops'" once torch pulls Triton 3.x;
#   - transformers too NEW for trl/peft -> "cannot import LossKwargs from transformers.utils"
#     (a newer transformers moved a symbol the trainer libs still import).
# So: keep transformers+trl as a COHERENT contemporaneous pair (capped), and let bitsandbytes
# float up (its newer releases are the ones that dropped the triton.ops import). torch is
# installed unpinned above (tracks your Python/driver). For strict reproducibility (spec FT-1),
# freeze the combo that works once training runs: `train/.venv/bin/pip freeze > train/requirements.lock`.

peft>=0.14
